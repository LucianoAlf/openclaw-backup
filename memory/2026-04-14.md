## Pluggy / Personal Finance LA — contexto de Phase 4

- MeuPluggy é gratuito de forma permanente para acessar dados próprios via Open Finance; o que expira após trial é webhook/auto-sync avançado do dashboard, não a API de leitura.
- Estratégia aprovada para Phase 4: polling via Supabase Edge Function (`poll-pluggy`) + `pg_cron` a cada 30–60 min, em vez de depender de webhook pago.
- Credenciais/itens do Pluggy foram organizados no `.env` do workspace como `PLUGGY_CLIENT_ID`, `PLUGGY_CLIENT_SECRET`, `PLUGGY_API_KEY` e `PLUGGY_ITEM_ID_*`.
- Testes reais na API Pluggy confirmaram:
  - autenticação via `POST /auth` com `clientId` e `clientSecret`
  - uso do header `X-API-KEY`
  - 5 items conectados e respondendo `UPDATED` + `SUCCESS`
  - leitura de contas/cartões em Santander, Mercado Pago, C6, Itaú e Nubank
  - leitura de investimentos reais no Nubank
  - leitura de transações reais em contas e cartões
  - `webhookUrl` ainda nulo nos items
- Direção de produto consolidada: Phase 4 = Central de Conciliação / Reconciliação Bancária, como superfície dedicada, não só extensão de cards.
- Estratégia de rollout aprovada: v1 com entrada manual + v1.5 com Pluggy/Open Finance, mantendo arquitetura source-agnostic.
- Documento recebido com arquitetura sugerida para Personal Finance LA cobre:
  - tabelas `pluggy_transactions`, `pluggy_accounts`, `pluggy_sync_log`
  - edge function `poll-pluggy`
  - auto-categorização inicial por regras
  - página `/conciliacao` com filtros, inbox de divergências, sync manual e conciliação inteligente.
